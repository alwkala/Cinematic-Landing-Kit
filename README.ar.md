# حزمة الصفحات السينمائية — Cinematic Landing Kit

حزمة سياق وذاكرة تُوجّه **أي وكيل ذكاء اصطناعي للبرمجة** لإنتاج صفحات هبوط سينمائية تفاعلية مبنية على التمرير — من النوع الذي يبدو كإعلان فاخر، وليس مجرد موقع ويب. ملف HTML واحد، مكتبات CDN، بدون خطوة بناء.

```
الوكيل يقرأ AGENTS.md  ←  يتبع memory/  ←  ينتج index.html  ←  انتهى.
```

يعمل مع أي وكيل ذكاء اصطناعي يقرأ ملفات التعليمات على مستوى المشروع (`AGENTS.md`، `.cursorrules`، system prompts، إلخ).

### الوكلاء المتوافقون

| وكيل | وكيل | وكيل |
|------|------|------|
| **Claude Code** | **Codex** | **Cursor** |
| **Kilo** | **Windsurf** | **Roo** |
| **Continue** | **Cline** | **OpenCode** |
| **Gemini CLI** | **OpenHands** | **Google Antigravity** |

> لا ترى وكيلك؟ إذا كان وكيلك يقرأ `AGENTS.md` من جذر المشروع، فهو يعمل. ضع الحزمة في مشروعك وابدأ.

## لماذا تفشل وكلاء الذكاء الاصطناعي

معظم وكلاء البرمجة بالذكاء الاصطناعي يمكنهم توليد صفحة هبوط.

قليلون جداً من يستطيعون إنتاج تجربة سينمائية بأسلوب Apple.

حزمة الصفحات السينمائية تُعلّم أي وكيل كيف يبني أفلام منتجات تفاعلية باستخدام أنماط مجربة في الإنتاج بدلاً من القوالب العامة.

| بدون هذه الحزمة | مع هذه الحزمة |
|-----------------|---------------|
| ❌ يستخدم `video.currentTime` (تقطيع) | ✅ فيلم إطارات على Canvas |
| ❌ خلل `mix-blend-mode` تحت GSAP | ✅ صور مقطوعة بشفافية PNG |
| ❌ انتقالات مكسورة (تلاشي أشباح) | ✅ مقاطع متطابقة الحدود |
| ❌ خطوط عشوائية (Amiri، خطوط النظام) | ✅ El Messiri + Tajawal + Cormorant |
| ❌ أصول غير متسقة أو وهمية | ✅ خط أنابيب متعدد المزودين (Qwen/Wan · Higgsfield · Nano Banana) |
| ❌ لغز `requestAnimationFrame` في التبويب المخفي | ✅ سير تحقق عبر `eval` |

### تقليدي مقابل سينمائي

| | صفحة هبوط تقليدية | حزمة الصفحات السينمائية |
|---|---|---|
| **البطل** | صورة ثابتة + تلاشي | قصة تُروى بالتمرير |
| **الحركة** | انتقال CSS `opacity` | فيلم Canvas مربوط بالتمرير |
| **التمرير** | سلبي، يكشف المحتوى فقط | موجَّه، حركة سينمائية |
| **الإحساس** | صفحة ثابتة | رحلة سردية متواصلة |
| **النتيجة** | عام، يشبه كل المواقع | فاخر — أسلوب Apple × Cartier |
| **الأصول** | صور مخزنية، تدرجات وهمية | مولَّدة بالذكاء الاصطناعي + قص rembg |
| **البناء** | إطار عمل + أدوات بناء + تبعيات | ملف HTML واحد، CDN فقط، صفر بناء |

## البداية السريعة

```bash
# ١ — انسخ الحزمة إلى جذر مشروعك:
cp -r AGENTS.md memory/ templates/ scripts/   /path/to/your-project/

# ٢ — أخبر وكيل الذكاء الاصطناعي:
"ابنِ صفحة هبوط سينمائية فاخرة لـ [المنتج].
  الصور المرجعية في assets/. اتبع AGENTS.md."

# ٣ — المعاينة:
python -m http.server 8123
# أو افتح index.html مباشرة في المتصفح
```

هذا كل شيء. الوكيل يقرأ `AGENTS.md`، يتبع ترتيب البناء، وينتج ملف `index.html` يعمل من المحاولة الأولى.

## المتطلبات

صفحة الهبوط نفسها **لا تحتاج أي متطلبات** — هي ملف HTML واحد بدون بناء، يمكن فتحه مباشرة في أي متصفح حديث أو تقديمه عبر أي خادم ثابت (مثل `python -m http.server 8123`).

أدوات Python وأدوات CLI هي **مساعدات اختيارية بالكامل** تُستخدم لتوليد أو استخراج أو تحسين أصول الوسائط:

| الأداة / الحزمة | الغرض | التثبيت |
|---|---|---|
| Python 3.8+ | تشغيل سكربتات المساعدة | أي إصدار 3.8+ |
| `rembg` | قص الخلفيات بشفافية | `pip install rembg` |
| `Pillow` | تغيير حجم / تحويل الصور | `pip install Pillow` |
| `opencv-python` | استخراج الإطارات | `pip install opencv-python` |
| `dashscope` *(اختياري)* | استدعاءات Qwen/Wan API | `pip install dashscope` |
| `higgsfield` CLI *(اختياري)* | توليد وسائط Higgsfield | `higgsfield auth login` |

تثبيت سكربتات المساعدة دفعة واحدة:

```bash
pip install rembg Pillow opencv-python dashscope
```

## المكدس التقني

المخرج هو **ملف `index.html` واحد** — بدون بناء، بدون أدوات تجميع، بدون إطار عمل.

| الطبقة | التقنية | المصدر |
|--------|---------|--------|
| محرك التمرير | Lenis 1.0 | CDN |
| الحركة | GSAP 3.12 + ScrollTrigger | CDN |
| التنسيق | Tailwind CSS | CDN |
| الخطوط | Google Fonts (El Messiri, Tajawal) | CDN |
| الفيلم | `<canvas>` + تسلسل إطارات JPG | أصول محلية |
| قص البطل | PNG شفاف (rembg) | أصول محلية |

الأصول البصرية تُولَّد عبر المزود الذي تختاره: **Nano Banana** (أداة `generate_image`)، أو **Qwen Image + Wan** (DashScope API)، أو **Higgsfield CLI**. يمكن استبدال أي نموذج مكافئ.

## القوالب وحالات الاستخدام المدعومة

خمسة أشكال من القوالب تغطي ١٥ حالة استخدام مختلفة. الوكيل يقرأ دليل التوجيه (`memory/10-use-cases.md`) لاختيار القالب المناسب تلقائياً.

| القالب | معالجة البطل | الفيلم | حالات الاستخدام |
|--------|-------------|--------|----------------|
| `fullbleed` | منتج مقطوع + هالة + جسيمات | فيلم تحول طويل (640vh) | إطلاق منتجات، مبيعات فاخرة، إعادة علامة تجارية، سيارات، أزياء، حِرَف، إصدارات محدودة |
| `editorial` | شاشة مقسمة (صورة + نص) | فيلم أقصر (420vh) | قصص علامات تجارية، فعاليات ومؤتمرات، صفحات مؤسسين |
| `spatial` | لقطة تأسيسية كاملة | فيلم جولة مكانية (500vh) | عقارات، عمارة، سفر فاخر، ضيافة |
| `interface` | واجهة جهاز (إطار CSS) | فيلم تدفق UI (420vh) | إطلاق SaaS، تطبيقات، منصات رقمية |
| `minimal` | بطل مركزي، بدون هالة | بدون فيلم | علامات شخصية، صناع محتوى، منتجات رقمية |

**اختبار الملاءمة:** موضوع واحد واضح + قوس تحول + المستخدم في وضع الإلهام ← الحزمة السينمائية.
كتالوجات متعددة المنتجات، صفحات مقارنة المواصفات ← صفحة هبوط تقليدية.

## brand.json — المصدر الوحيد للحقيقة

هوية العلامة التجارية لكل مشروع تعيش في **`brand.json`** في جذر المشروع. عندما يكون موجوداً، يقرأه الوكيل **قبل** تشكيل القالب، وتتجاوز رموزه جميع الافتراضيات. الحزمة تتكيف بالكامل مع العلامة التجارية — غيّر `primary` من الذهبي إلى الأزرق، وتتلون التجربة السينمائية بالكامل مع الحفاظ على بنية الحركة.

### ما يتحكم به

| مجموعة الرموز | سلوك الحزمة |
|---------------|------------|
| `meta.*` | يملأ `{{PRODUCT}}`، `<title>`، `<meta description>`، علامات OG |
| `colors.light.*` / `colors.dark.*` | يُربط بمتغيرات CSS الجذرية (`--paper`، `--ink`، `--gold`، إلخ) |
| `voice.*` | يحكم كل نص مولَّد — تعليقات، أزرار، عناوين فرعية. الكلمات في `voice.doNotUse` محظورة |
| `identity.logo.*` | مسارات الأيقونة، شعار التنقل، صورة OG |
| `typography.families.*` | خطوط العناوين والنص. الخط اللاتيني يبقى ما لم يُتجاوز |
| `localization.*` | يضبط `<html lang>` و `<html dir>` لعرض RTL/LTR |
| `motion.*` | منحنيات التسهيل وسلوك الحركة المخفضة |
| `accessibility.*` | حلقة التركيز، هدف اللمس، سياسات النص البديل |

ربط الرموز الكامل بمتغيرات CSS: `memory/11-brand-json.md`.

### لماذا brand.json مهم

- **قابلية إعادة الإنتاج** — أعد توليد أي صفحة من نفس brand.json وتحصل على نتائج متطابقة
- **قابلية النقل** — نفس `brand.json` يخدم الحزمة السينمائية وحزمة التوثيق والحزم المستقبلية
- **بدون قيم ثابتة** — الوكلاء ممنوعون صراحة من تضمين قيمة لون أو اسم خط له مكافئ في brand.json
- **اتساق النبرة** — كل تعليق وزر وعنوان فرعي يُكتب بأسلوب العلامة التجارية

ملف `brand.json` بسيط (أقل من ٥٠ سطراً) يستغرق حوالي ٣ دقائق لكتابته ويزيل كل انحراف نحو "الذهبي الافتراضي" في صفحات الذكاء الاصطناعي.

## ما يعرفه الوكيل قبل أن يبدأ

هذه هي الدروس المُكتسبة المُشفَّرة في `memory/` — الأشياء التي يخطئ فيها الوكلاء عادةً في المحاولة الأولى:

- **اقرأ `brand.json` أولاً** عند وجوده — ألوانه وخطوطه ونبرته وملفات هويته تتجاوز كل افتراضيات القالب
- **لا تستخدم `video.currentTime` أبداً** — فيلم التمرير هو تسلسل إطارات Canvas، وليس عنصر فيديو
- **لا تستخدم `mix-blend-mode`** على العناصر المتحركة — يتعطل تحت تحويلات GSAP. استخدم صور PNG شفافة
- **لا تمزج صورتين ثابتتين بالتلاشي** — الانتقالات السلسة تتطلب مقاطع فيديو متطابقة الحدود
- **زامن `FRAME_COUNT`** في HTML مع العدد الفعلي للإطارات المستخرجة
- **التبويبات المخفية توقف `requestAnimationFrame`** — تحقق عبر `eval`، وليس لقطات الشاشة
- **الخطوط العربية:** El Messiri (عناوين) + Tajawal (نص). **أبداً Amiri**
- الضوابط إلزامية لأي موضوع بشري. هوية المنتج تُحفظ بالضبط عبر جميع الأصول المولَّدة
- أبطال حسب القالب: `fullbleed`/`editorial`/`minimal` تستخدم صورة PNG شفافة مقطوعة. `spatial` تستخدم لقطة تأسيسية كاملة. `interface` تستخدم إطار واجهة CSS
